1"""Interface DataSource e tipos relacionados — Camada 4 Parte B.1.a. 

2 

3DataSource é o ponto de extensão para adicionar novas fontes de dados ao 

4AlertaVida. Cada implementação concreta encapsula: 

5- Transporte (HTTP, FTP, arquivo, etc.) 

6- Parsing do payload bruto 

7- Mapeamento para o domínio Alerta (incluindo classificação COBRADE 

8 e cálculo de escopo geográfico) 

9- Retry/backoff específico da fonte 

10 

11A interface NÃO expõe o transporte usado — subclasses podem usar urllib, 

12httpx, ler de disco, etc. Isso permite substituição em testes via 

13subclasse-fake (FakeDataSource em tests/fixtures/sources_fake.py), 

14sem precisar mockar bibliotecas de rede. 

15 

16Design decisions registradas em wiki/decisions/datasource-adapter-falha-de-coleta.md: 

17- DataSource como ABC com ResultadoColeta tipado, não list[Alerta] 

18- FalhaDeColeta como exceção tipada do domínio 

19- Síncrono (não async) — Camada 6 pode usar run_in_threadpool se necessário 

20""" 

21

32@dataclass(frozen=True) 

33class ResultadoColeta: 

34 """Resultado de uma rodada de coleta de uma DataSource. 

35 

36 Frozen — imutável após construção. Carrega contabilidade explícita 

37 (alertas válidos + contagem de descartes) para que o orquestrador 

38 preserve a assertion sanitária por fonte: 

39 coletados == novos + atualizados + inalterados + descartados + erros 

40 

41 Sem `descartados`, o orquestrador teria que adivinhar quantos itens 

42 a source filtrou — informação que apenas a source tem. 

43 

44 Atributos: 

45 alertas: lista de Alertas já validados pelo Pydantic, prontos 

46 para o pipeline detector → database. Cada Alerta carrega 

47 fonte=DataSource.fonte populada pela source. 

48 descartados: número de itens brutos que a source filtrou por 

49 malformação (campos obrigatórios ausentes, tipos inválidos). 

50 Logging interno é responsabilidade da source. 

51 coletado_em: timestamp aware (com tzinfo) do momento em que a 

52 coleta foi concluída. Usado pelo orquestrador em logs e 

53 relatórios. 

54 """ 

55 

56 alertas: list[Alerta] 

57 descartados: int 

58 coletado_em: datetime

61class FalhaDeColeta(Exception): 

62 """Falha irrecuperável durante coleta de uma DataSource. 

63 

64 A source levanta isso após esgotar retries internos (rede caída, 

65 schema rejeitado pela response, JSON corrompido). O orquestrador 

66 (Parte B.2) captura `FalhaDeColeta`, registra a falha contra a 

67 fonte específica via RelatorioFonte(falha_coleta=True), e continua 

68 com as próximas fontes na mesma rodada. 

69 

70 Falhas INDIVIDUAIS de alerta dentro de uma rodada NÃO sobem como 

71 FalhaDeColeta — são contadas em ResultadoColeta.descartados. 

72 FalhaDeColeta é reservado para falhas de RODADA INTEIRA. 

73 

74 Atributos: 

75 fonte: qual DataSource originou a falha. Permite ao orquestrador 

76 identificar a fonte sem rastrear contexto externo. 

77 causa: descrição curta legível (ex: "rede esgotada após 4 tentativas", 

78 "schema rejeitado: JSON inválido"). 

79 original: exceção original que causou a falha (URLError, HTTPError, 

80 JSONDecodeError, etc.). Preservada via exception chain para 

81 debug — sem isso, perde-se o stack trace original. 

82 

83 Uso típico em uma implementação: 

84 try: 

85 raw = self._fetch_com_retry() 

86 except (URLError, socket.timeout) as exc: 

87 raise FalhaDeColeta( 

88 fonte=self.fonte, 

89 causa="rede esgotada após retries", 

90 original=exc, 

91 ) from exc 

92 """ 

93 

94 def __init__( 

95 self, 

96 fonte: FonteDado, 

97 causa: str, 

98 original: Exception | None = None, 

99 ) -> None: 

100 self.fonte = fonte 

101 self.causa = causa 

102 self.original = original 

103 super().__init__(f"Falha de coleta em {fonte.value}: {causa}")

106class DataSource(ABC): 

107 """Interface comum para fontes de dados de alertas. 

108 

109 Cada implementação concreta encapsula tudo que é específico de uma 

110 fonte: transporte, parsing, mapeamento de tipos para COBRADE, 

111 classificação geográfica, retry/backoff. 

112 

113 Contrato: 

114 - `fonte` é estável durante o ciclo de vida da instância. Não muda. 

115 - `coletar()` é síncrono. Faz I/O de rede de LEITURA apenas. 

116 - `coletar()` NÃO escreve em banco, NÃO imprime (usa logging), NÃO 

117 modifica filesystem. Pureza de side effects exceto rede. 

118 - Falhas individuais de alerta são contadas em 

119 ResultadoColeta.descartados; não sobem como exceção. 

120 - Falha de rodada inteira sobe como FalhaDeColeta; orquestrador 

121 captura e continua. 

122 - Qualquer outra exceção é bug e propaga (regra de 

123 wiki/patterns/resilience-invariants.md: nunca bare except). 

124 """ 

125 

126 @property 

127 @abstractmethod 

128 def fonte(self) -> FonteDado: 

129 """Identificador estável da fonte. Vai para Alerta.fonte e 

130 para a coluna `fonte` no banco. Property abstrata força 

131 declaração explícita pela subclasse. 

132 """ 

133 

134 @abstractmethod 

135 def coletar(self) -> ResultadoColeta: 

136 """Executa uma rodada completa de coleta. 

137 

138 Retorna ResultadoColeta com Alertas já validados e enriquecidos 

139 (cobrade_codigo, escopo_geografico, fonte_classificacao populados 

140 pela source). Levanta FalhaDeColeta em falha de rodada. 

141 """
